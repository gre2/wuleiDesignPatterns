设计模式

```
动态代理
1.静态代理；动态代理（jdk,cglib）；手写代理（字节码重组）
2.两个角色：代理人，被代理人
3.对于被代理人来说，事情一定要做，自己不想或者没时间做，找代理
4.代理人必须要获取到被代理人的引用
5.关心过程，不关心结果
6.归纳：办事要求人。所以找代理
```

```
手写代理（字节码重组）
1.生成源代码
2.将生成的源代码保存为java文件
3.编译源代码，并生成class文件
4.将class文件的内容，动态加载到jvm中
5.返回被代理后的代理对象
```

```
工厂模式
1.对于调用者来说，隐藏了复杂的逻辑处理过程，调用者只关心执行结果
2.对于工厂来说，对结果负责，保证生产出符合规范的产品
3.归纳：只对结果负责，不要三无产品
```

```
单例
1.一个jvm生命周期，唯一，只有一个实例
```

```
委派模式
1.两个角色：委拖人和被委托人
2.权力上平等（实现相同的接口）
3.委托人持有被委托人的引用
4.不关系过程，只关心结果
5.干活是你的（员工），功劳是我的（经理）
```

```
策略模式
1.执行逻辑不一样，最终执行结果是固定的
2.归纳：我行我素，达到目的就行
```

```
原型模式
1.数据内容相同，但对象实例不同
2.归纳：拔一根猴毛，吹出千万个
```

```
模板模式
1.执行流程固定，但中间有些步骤有细微差别(运行时确定)
2.可以实现批量生产
3.归纳：流程标准化，原料自己加
```
